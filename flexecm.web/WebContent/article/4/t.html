创建操作栏

本节我们在H5方面将实现一个平滑切换的操作栏，在操作栏上提供对文件列表的增加删除修改等基本操作


操作栏草图如下

文件列表  页面剩余部分
  +                       （）

首先我在body中增加
<div class="btnbar">
    <div class="normal flexbox">

        <a class="item">刷新</a>

</div>
将它绝对定位到整个页面底部。

让操作按钮平均分布

我们希望让创建和刷新按钮进行水平平均分布。 
我们利用HTML5的flexs box 模型，它的标准是 http://www.w3.org/TR/css3-flexbox/
flex box这个协议本身是为了重新定义css的布局方式。 因为自有HTML以来， 布局一直是一个老大难问题，最早是用table进行布局，后来是用div布局，现在到了移动设备上，更加困难。我们之前讲过使用的是div  position absolute方式进行，其实也可以考虑以后用flex box形式。

我们甚至可以想象的到未来大家可能都会用这个模型来进行页面布局了。 当然这也要随着设备和客户端的升级和大家的认识。 

这里有几篇不错的文章来讲述， 我们不详细描述。现在只是用这个特性来进行水平平均分布的实现
http://zh.learnlayout.com/flexbox.html 
https://css-tricks.com/snippets/css/a-guide-to-flexbox/

首先在外围容器上写
.flexbox {
 display: -webkit-box;
 display: -moz-box;
 display: -ms-flexbox;
 display: -webkit-flex;
 display: flex;
}
在内部的item写
.flexbox .item {
 -webkit-box-flex: 1;
 -moz-box-flex: 1;
 -webkit-flex: 1;
 -ms-flex: 1;
 flex: 1;
 text-align: center;
}

因为flexbox的标准发生过变动，因此安卓2.1开始和4.x版本可能写法不同。 上面的写法应该是兼容2种标准的


用webfont来实现按钮图标

我们需要在操作按钮上显示图标，这里我推荐用webfont方式。对于图标，大家可能要说：这还不容易，但是要知道在移动设备上，对图标的要求是它的大小是可变的，webfont实际上是我们可以用图标来作为字符的显示。 最近的项目我都用webfont来实现图标，效果很好，这里也推荐大家使用

1 前往 http://fontello.com/  
2 挑选需要的图标
3 打包下载

下载后打开大概是类似这样的一个目录 
非常好， 把css和font目录拷贝到你的工作目录下吧。  注意里面的文件不要和你自己的相同

4 在你的页面中首部加入
<link rel="stylesheet" href="css/fontello.css">

5 然后给你需要加图标的元素增加类吧，类名是什么呢， 打开上图中的demo.html

给操作栏增加特效

在本应用中，我们给每个文件前面增加一个复选框。当用户点击复选框时，操作栏的按钮会变为移动、复制和删除，同时，新建和刷新按钮隐藏。这需要我们在btnbar下面另建一个flexbox。 它的类是selected

 <div class="flexbox selected">
     <a class="item icon-trash">删除</a>
     <a class="item icon-th-large-outline">复制</a>
     <a class="item icon-right">移动</a>
    </div>   

当然我们可以用最简单的方式就是
$(".btnbar .normal").hide();
$(".btnbar .selected").show();

但这样不够酷。 我们希望新的操作栏向上滑动。好，我们现在实现这个功能。
方案1
首先我们用修改margin-top（上对齐）的方式 
$(".btnbar .selected").css("margin-top", "0");
$(".btnbar .normal").css("margin-top", "3rem");

同时， 我们为他们增加一段css
.btnbar div.flexbox {
    -webkit-transition: margin-top .3s ease-in-out;
}
css的意思就是说当margin-top变动时， 我们使用一个过渡效果ease-in-out，  效果持续时间是0.3秒
这段代码很常见， 在PC中自不必说。在手机上表现也可以，但是如果你仔细观察，动画效果还不是很流畅，加上 -webkit-transform: translate3d(0,0,0); 也没有用。 还记得第二节我们讲到的几个浏览器廉价渲染方案么？ 这就是我们方案2 

方案2

首先
我们为这些flexbox容器加上大小属性，让它们绝对定位
 position:absolute;
 bottom: 0;
 top: 0;
 left: 0;
 right: 0;

代码中
$(".btnbar .selected").css("transform", "translate(0, 0) translateZ(0px)");
 $(".btnbar .normal").css("transform", "translate(0, 3rem) translateZ(0px)")
transform: translate是说让目标元素在原有位置进行进一步x、y、z的变换。   
然后给flexbox在加上变换效果
 -webkit-transition-duration: 200ms;
-webkit-transform: translateZ(0px);
最关键的是   -webkit-transform: translateZ(0px); 这个属性。 因为它要求进行z变幻，而浏览器的默认渲染是无法支持的，所以浏览器调用了GPU进行处理，虽然z:0等于没做什么，但GPU一旦加入，速度就得到了极大提升。
-webkit-transform: translate3d(0,0,0);  这么写也是这个目的，然而，在移动端 这个写法很可能只对translate(x, y)生效，对像margin-top这样的css变化似乎并无效果。所以在移动端首选方案2的形式。 速度的确快了很多，媲美原生。

切换触发

最后，我们给每个文件、文件夹前面增加一个checkbox， 当有checkbox被选中时，操作栏切换为编辑模式 

cloned.find(".check").attach(function() {
       if ($(this).hasClass("icon-check")) {
          $(this).removeClass("icon-check");
          $(this).addClass("icon-check-empty");
       } else {
          $(this).addClass("icon-check");
          $(this).removeClass("icon-check-empty");
       }
       if ($(".check.icon-check").length>0) {
           editMode();
        } else {
           listMode();
       }
   }, true);

值得注意的是 attach方法，我们新增了第二个参数是true 。 
表示事件不向上传播。  因为我们对目录的整个li增加了事件

 cloned.attach(function() {
     openDir($(this).data("entry"));
  });

如果传播，整个目录被打开，那么操作流程就完全乱了。  


小结 
 本章我们围绕操作栏及特效进行了讲解。核心还是介绍一些H5的常用特性， 包括flexbox布局、webfont图标、translate制作动态效果等。这些都是常用而且有效的功能。
