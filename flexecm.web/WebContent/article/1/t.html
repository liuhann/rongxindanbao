Phonegap介绍、安装和基本概念

作者：寒流 分类 ： 学习HTML5 cordova教程

创建时间： 2015-3-4 16:01 修改时间： 2015-3-9 11:23
    Phonegap简介

    phonegap是一个移动开发框架。它将HTML、CSS、JS封装为一个手机app。这种开发模式通常称为 hybrid（混血）。因为它既不同于原生的手机app，又与纯HTML应用不完全一样。 在phonegap中， 移动设备上的很多功能都被封装为H5 JS里面可以调用的对象。

    说起phonegap，人们往往想到了Java。Java诞生时有一个很响亮的口号，叫做write once， run everywhere。 意思是一次编写，到处运行。想法是很好的，但后来的发展结果是write once、debug everwhere。 一次编写、到处调试。 虽然Java的口号没有走多远，但Java本身却走了很远，到现在安卓的开发使用的语言就是Java，Java在服务器端也得到了最广泛的使用。 phonegap出来的时候，他的想法也是类似： 通过封装使得编写好的应用可以部署到 安卓、iOS、WP甚至BB10等移动设备上。  基于这个想法，很多人认为phonegap可以用来节省成本，可以少雇一名程序员。但就我的经验，如果出于节省成本的考虑就算了。phonegap妥妥地会一次编写、到处调试，phonegap所依赖的HTML5还是很靠谱的。在学习phonegap之前，你可能在网上看到很多负面评论，这些评论很多是真的，也有很多已经是历史问题了。 不管怎样，HTML5在移动设备上的表现也是在日益成熟中。 

下面是citepoiont的一篇文章， Do You Really Need an App for That?  http://www.sitepoint.com/really-need-app/ 

作者的观点很鲜明，在APP领域，文档化的应用会以纯HTML形式提供， 而行为化的会逐渐有Hybrid形式。 

    

    phonegap经常和Cordova混说在一起。 phonegap是一个公司的产品，2011年 adobe收购了这个公司，然后把phonegap的主要工作开源贡献给了apache， 成立了开源项目apache cordova， 现在说cordova和phonegap也基本不做区分了。
   我认为通常开源化有3个主要目的：  
        贡献知识，共同发展
        消除疑虑，促进商业应用
        形成开源壁垒，阻止竞争对手
因此cordova设立后，不光促进了hybrid应用的发展，phonegap本身也不断在升级完善，商业上也逐步被应用，在这个领域也确立了领先的地位。
学习cordova主要参考资料就是：
cordova 首页 http://cordova.apache.org/  
还有文档地址  http://cordova.apache.org/docs/en/4.0.0/
    因此，我们学习phonegap/cordova，可以对HTML5进行一次更完整的认知。未来移动应用形式中， Hybrid类型会使用的越来越广泛，也可能是未来一个成熟的方向。
    
安装cordova  

1 安装nodejs 

2 安装cordova 

  命令行执行 npm install cordova  如果npm运行不成功，请确定一下nodejs目录在PATH环境变量中

3 安装Java JDK  Android SDK 
 
  安装安卓SDK可能要越GFW   安装完成后还要确认 Java的bin目录及android sdk的 platform-tools、tools目录都在PATH环境变量中

4 创建helloword项目 

cordova create hello com.example.hello HelloWorld
cd hello
cordova platform add android
cordova build


这几步是创建一个HelloWorld项目， 放在hello目录里面 。 然后添加了andriod运行时，并编译。 如果第三步Java 或andriod SDK没有正确设置的化， 添加android运行时就可能失败。

现在我们简单扫一眼以上几个命令为我们建立的工程

 包含了4个目录和一个config.xml配置文件。 

config.xml 是cordova的核心配置信息。 目前看应该是这样子的


<?xml version='1.0' encoding='utf-8'?>
<widget id="com.example.hello" version="0.0.1" xmlns="http://www.w3.org/ns/widgets" xmlns:cdv="http://cordova.apache.org/ns/1.0">
    <name>HelloWorld</name>
    <description>
        A sample Apache Cordova application that responds to the deviceready event.
    </description>
    <author email="dev@cordova.apache.org" href="http://cordova.io">
        Apache Cordova Team
    </author>
    <content src="index.html" />
    <access origin="*" />
</widget>


它是一个符合w3 widget定义的一个配置文件，如果你的应用更偏向于native的话，可能要经常修改它。 
另外还有四个目录，其中
hook 存放的是自定义扩展功能。 
platform 存放添加的运行时
plugins 存放引入的插件
www 则是我们的开发目录， 我们开发的H5相关HTML、JS、CSS、图片资源都放到这个目录。

选择刚才的hello 目录 



在这里你会看到2个项目，另一个是cordova的lib，如果你第一次导入项目，要把它也导入。 如果是之后， 则不需要再次导入

点finish。 或许项目还有报错， 您可以点菜单project->clean清除一下



如果还报错，可能是您导入了第二个项目， 没有引入CordovaLib，这里您需要到project属性里面找到andriod属性栏，

3 实现一个简单的功能


现在我们要简单实现一个功能， 把DEVICE IS READY部分的DEVICE改为你的手机型号。 

这里我们要使用cordova提供的一个JS对象 device。cordova提供的所有对象都是以插件形式提供的(这和早期版本不同)，
我们先在命令行执行 

cordova plugin add org.apache.cordova.device

然后打开js/index.js ,插入以下行

   receivedEvent: function(id) {
        var parentElement = document.getElementById(id);
        var listeningElement = parentElement.querySelector('.listening');
        var receivedElement = parentElement.querySelector('.received');

        listeningElement.setAttribute('style', 'display:none;');
        receivedElement.setAttribute('style', 'display:block;');
       
        receivedElement.innerHTML = "Hello" + device.model;  
       
        console.log('Received Event: ' + id);
    }




4 cordova基本概念和加载次序

cordova的主要功能是以Event 事件和plugin 插件的形式提供的。

事件就是指像应用被转到后台、被唤醒等情况下，用户可以采取的处理。cordova提供了以下事件， 根据设备的不同，支持的事件也不同

    - deviceready
    - pause
    - resume
    - backbutton
    - menubutton
    - searchbutton
    - startcallbutton
    - endcallbutton
    - volumedownbutton
    - volumeupbutton

而插件则是以JS对象形式提供的，通过调用这些对象可以去调用手机原生的一些功能，比如文件系统、摄像头、GPS等信息。

cordova最重要的一个事件是 deviceready 。 必须在监听到deviceready 之后，各种插件才能被正确的调用。
一个基于cordova的应用启动过程如下




因此我们可能要写类似这样的代码：
document.addEventListener('deviceready', this.onDeviceReady, false);

function onDeviceReady() {
     $(document).ready(function() {
        //你的代码从这里开始
     });
};
   
在onDeviceReady里面再进一步$(document).ready。或者在 $(document).ready只执行UI、事件绑定等处理。 在onDeviceReady里面再做涉及到设备的相关操作。
