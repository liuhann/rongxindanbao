为应用构建一个基本的列表页面

作者：寒流 分类 ： 学习HTML5 cordova教程

创建时间： 2015-3-5 11:34 修改时间： 2015-3-9 11:24
HTML5在移动开发中的一些准则

1  尝试将所有东西写在一个页面
    虽然看起来复杂一些，但多页面的应用会有很多问题，首先切换特效就别考虑了，然后如果在web上用户加载页面的话有可能会等比较长时间，用户体验中断。如果新页面加载失败了，你没办法去控制进一步的处理，只能让用户点击回退，最后页面之间跳转、数据重新加载等问题也够喝一壶的。

2 尽量不用重量级的前端JS框架。 特别是面向UI的框架。
  Jquery mobile我在alpha期间就用过， 虽然能快速开发出一些效果，然而面对新需求或者UI本身有坑时，你就知道痛苦了。效率先不考虑，UI问题千变万化，还是自己做着放心。  jquery是必须要用的，而且要用jquery 2以上的版本，它摆脱了IE的束缚

3 引入动画、特效或者一些特定的css要慎重。 
按照 这篇文章来说 
http://www.html5rocks.com/en/tutorials/speed/high-performance-animations/
以下四个css的特效对是最快速高效的，它们会直接调用GPU， 除此之外的动画方案相应都是高代价的。


在本系列后面部分会使用以上特性制作动态效果 。

目前web特效的标准草案还是制定中， 在以后的移动端特效在新设备上会越来越好
 http://www.w3.org/TR/web-animations/


4 使用position absolute避免页面滚动条

5 长度单位请使用rem
关于 px  em  rem这里不再多说


页面基本布局构成

首先，我们的应用总体布局如下

标题栏    高度3rem
文件路径   高度 2.5rem
文件列表  页面剩余部分

我们先对标题栏、路径栏、文件列表进行制作

代码

  <div class="header">
       叮咚文件管理器
  </div>
  <div class="nav">
   
  </div>
   
  <div class="list">
       
  </div>

我们构建了3个div  他们使用的定位分别是

.header {
     position: absolute;
     top: 0;
     left: 0;
     right: 0;
     height: 2.5rem;
}

. nav {
   position: absolute;
   top: 2.5rem;
   left: 0;
  right: 0;
  height: 2rem;
}

.list {
 position: absolute;
 top: 5rem;
 bottom: 3rem;
 left: 0;
 right: 0;
 overflow: hidden;
}



这样就完成了页面基本布局

用iscrolljs实现列表

上一节里面， 我们给列表定义了一个相对屏幕的固定大小， 然而列表长度是可变的， 在移动开发中，一个可变的、能平滑滚动的列表组件是必须的。这里我们介绍大家使用 iscrolljs。 


它的首页是  http://iscrolljs.com/

isrolljs发展了很多年，因为其轻便、小巧，现在在移动端也得到不少应用。

iscrolljs有数个版本，应用于不同的场合。在手机上 推荐下载iscroll-lite.js版本


首先我们修改list内容 

<div class="list">
         <ul>
          <li>
                  内容1
          </li>
         </ul>
  </div>
        
然后加入代码
var listscroll = new IScroll('.list');

为了能看到滚动效果，我们可以这样
for (var i=0;i<300;i++) {
    $('.list ul').append('<li>很平滑把' + i + '</li>');
}


iscrolljs的使用， 大家要注意以下几点

1 初始化iscroll的容器必须是绝对或相对定位的  position:relative 、absolute 
2 同样容器必须是overflow: hidden; 不要给自己找trouble
3 为了性能， 滚动的内容里面最好不要设置以下的css内容  box-shadow, opacity, text-shadow 
实际上以上属性在位置会变化的元素里都不要设置。
4 当容器内容发生变化时，要调用 refresh()方法进行刷新。


构建类表内容

上一节构建列表时，列表的内容是一段文字， 而实际应用中，列表显示往往是很复杂的


首先， 我们将li的内容设置为

 <li class="item">
           <div class="check icon-check-empty"></div>
           <div class="icon"></div>
           <div class="name"></div>
  </li>

li的css设置为
.list ul li.item {
     position:relative;
     list-style: none;
     border-bottom: 1px solid #ddd;
     height: 3rem;
     overflow: hidden;
     white-space: nowrap;
     text-overflow:ellipsis;
}

而里面的3个div分别定位如下

.list ul li.item div.check {
 position: absolute;
 left: .2rem;
 font-size: 1.25rem;
 color: #ddd;
 padding: .6rem .4rem;
}
.list ul li.item div.icon {
 position: absolute;
 left: 2.5rem;
 height: 2.5rem;
 width: 2.5rem;
}
.list ul li.item div.name {
 position: absolute;
 top:.8rem;
 font-size: 1rem;
 height: 1.5rem;
 left: 5.5rem;
}

我们找到一个文件夹的图片， 为icon的div增加一个folder的class
.list ul li.item div.icon.folder {
 background-image: url("../img/folder.png");
 background-repeat: no-repeat;
 background-size: cover;
}

这样一个简单的列表就出来了， 不考虑最左面的checkbox 

实现点击和长按事件

在移动设备中，我们不能像电脑上一样， 写一个onclick事件。 很多情况下可能会没有反应，因为click是一个鼠标动作， 移动设备只能模拟，速度也变慢了， 也没有按压效果。 相反，我们可以用2个触碰事件来代替  touchstart和touchend

这段代码自用了很长时间， 具体思路就是同时增加 touchstart  和 touchend 2个事件监听。 当一个元素先后经历这2个事件时，就认为完成了一次tap，同时在touchstart  后为元素增加了一个css类 pressed。  当用户手移动时(监听touchmove)， touchend 就会失效。

https://github.com/liuhann/mobile-attach-event/blob/master/attach.js

长按事件思路也是类似， 就是touchstart  后一段时间还没有touchend  就认为进行了长按。



小结

本文主要介绍一些H5开发的基本准则，从实战出发，制作一个原型页面。 页面包括了应用布局、平滑的列表、点击事件处理等基本操作。
